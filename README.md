# Claude Code 교육 레포

대학교 동아리 세션에서 **Claude Code의 주요 기능을 직접 켜보면서 익히는** 실습 레포입니다. 입문자가 VSCode에 Claude Code 확장을 띄워놓고 발표를 따라가다가, 자기 화면에서 그대로 실행해볼 수 있도록 구성되어 있습니다.

## 이 레포로 무엇을 배우나요?

Claude Code는 단순히 "AI에게 코드를 시키는 CLI"가 아니라, 그 위에 **자동화·확장 기능**이 켜켜이 쌓여 있는 도구입니다. 이 레포는 그 기능들을 **출시된 순서대로**, 그리고 서로 어떻게 관계 맺는지를 한눈에 보여줍니다.

### 출시 타임라인

| 시기 | 기능 | 한 줄 요약 |
|---|---|---|
| 2024-11 | **MCP** | 외부 서비스(Jira, Slack, Drive 등)와 Claude를 연결하는 표준 프로토콜 |
| 2025-07 | **Sub-agent** | 격리된 컨텍스트에서 돌아가는 독립 AI 인스턴스 — 결과만 메인 세션으로 반환 |
| 2025-09 | **Hook** | 툴 호출 같은 라이프사이클 이벤트에 결정론적으로 개입하는 스크립트 |
| 2025-10 | **Plugin** | Skill·Hook·Agent 등을 묶어 팀/프로젝트 간 배포 가능한 패키지 |
| 2025-10 | **Skill** | 반복 워크플로우를 `SKILL.md`로 작성해두면 슬래시 커맨드나 자동 인식으로 호출 |
| 2026-02 | **Agent Team** | 여러 Claude 세션이 공유 태스크 리스트와 직접 메시지로 **양방향** 협력하는 구조 |

> 슬래시 커맨드(`/이름`)는 출시 초기부터 있던 인터페이스로, 2026-04에 Skill 시스템에 통합되었습니다. 이 레포에서는 Skill 데모 안에서 자연스럽게 다룹니다.

## 기능 간 관계 — "격리도" 스펙트럼

```
            [ 같은 컨텍스트 ]                    [ 격리된 컨텍스트 ]              [ 별도 프로세스 ]
                  │                                      │                               │
              Skills ─────────────────► Sub-agents ─────────────────► Agent Teams
              (즉시·가벼움)              (병렬·결과만 반환)              (양방향·복잡)

              ◄────────────────  Hook & Plugin은 모든 레벨에 적용  ────────────────►
```

오른쪽으로 갈수록 **격리도와 병렬성**이 높아지고, 동시에 **오버헤드**도 커집니다.

## 디렉토리 구조

```
.
├── README.md           ← 지금 보고 있는 파일
├── CLAUDE.md           ← Claude Code가 자동으로 읽는 프로젝트 설명
├── .claude/
│   ├── agents/         ← Sub-agent 정의
│   ├── skills/         ← Skill 정의 (SKILL.md)
│   └── settings.json   ← Hook 등 프로젝트 설정
├── demos/
│   ├── 01_skill/       ← /명령으로 호출되는 Skill 시연
│   ├── 02_subagent/    ← 격리된 컨텍스트에서 병렬 작업
│   ├── 03_hook/        ← 파일 저장 시 자동 실행되는 Hook
│   ├── 04_agent_team/  ← 여러 세션이 협력하는 Agent Team
│   └── 05_mcp/         ← 외부 서비스 연결 (MCP)
└── scripts/            ← 데모 환경 셋업 스크립트
```

## 권장 시연 흐름

1. **CLAUDE.md** — Claude Code가 프로젝트를 이해하는 방식 설명
2. **Skill** (`demos/01_skill`) — `/명령`을 호출해 즉각 반응하는 모습 시연
3. **Sub-agent** (`demos/02_subagent`) — 병렬 작업이 어떻게 분리·요약되는지
4. **Hook** (`demos/03_hook`) — 파일 저장 시 자동 실행되는 모습
5. **Agent Team** (`demos/04_agent_team`) — 가장 복잡한 기능으로 마무리
6. **MCP** (`demos/05_mcp`) — 외부 세계와의 연결 (시간이 남으면)

> **Plugin**은 별도 데모를 두지 않습니다. 5개 데모를 모두 본 뒤 "이걸 한 번에 묶어서 배포하면 그게 Plugin이다"라는 흐름으로 자연스럽게 설명하는 게 가장 직관적입니다.

## 시작하기

```bash
# 1) 레포 클론
git clone <이 레포 URL>
cd <레포 디렉토리>

# 2) VSCode에서 열기
code .

# 3) Claude Code 확장 실행 후, 각 demos/XX_*/README.md를 따라가세요
```

각 데모 디렉토리에는 그 기능 하나에만 집중한 `README.md`와 실제 동작하는 예제가 들어 있습니다.
