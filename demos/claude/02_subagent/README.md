# 02. Sub-agent 데모

> **Sub-agent란?** **격리된 컨텍스트**에서 돌아가는 독립 Claude 인스턴스입니다. 메인 세션이 작업을 위임하면, Sub-agent는 자기만의 컨텍스트에서 일하고 **요약된 결과만** 메인으로 돌려보냅니다. 메인 세션의 토큰을 보호하면서, 여러 Sub-agent를 **병렬로** 돌릴 수도 있습니다.

## Skill과의 차이

| | Skill | Sub-agent |
|---|---|---|
| 어디서 돌아가나 | **메인 세션과 같은** 컨텍스트 | **격리된 별도** 컨텍스트 |
| 메인 토큰 영향 | 모든 출력이 메인 토큰 사용 | 요약만 메인 토큰 사용 |
| 병렬 실행 | 불가 (한 번에 하나) | **가능** (여러 개 동시에) |
| 적합한 작업 | 짧은 반복 작업, 출력 포맷팅 | 검색·조사·검토처럼 **읽을 게 많은** 작업 |
| 정의 위치 | `.claude/skills/<이름>/SKILL.md` | `.claude/agents/<이름>.md` |

핵심 한 줄: **"메인 세션이 다 알 필요 없는 작업"** 을 Sub-agent에게 넘깁니다.

## 직접 써볼 Sub-agent 두 개

### A. [`explorer`](../../../.claude/agents/explorer.md) — 탐색 전용

레포 안에서 키워드·패턴·파일을 찾는 데 특화된 Sub-agent. 광범위한 grep/find/read를 자기 컨텍스트에서 수행하고, 메인 세션엔 **링크와 한 줄 설명만** 돌려줍니다.

### B. [`reviewer`](../../../.claude/agents/reviewer.md) — 코드 리뷰 전용

현재 git 변경사항을 자기 컨텍스트에서 읽고 리뷰 의견만 반환합니다. 메인 세션은 diff 원문을 보지 않아도 핵심을 파악할 수 있습니다.

## 직접 해보기

### 시도해보기 1 — 단일 호출로 "격리" 확인하기

1. 메인 채팅창에 입력:
   > **"explorer 에이전트로 이 레포에서 'Skill'이라는 단어가 어디서 쓰이는지 찾아봐줘"**
2. Claude Code UI에서 **Sub-agent 실행 표시**가 뜨는 것을 확인 (별도 진행 표시줄)
3. 결과는 짧은 요약 + 링크 목록으로 돌아옴
4. **여기서 알 수 있는 것:** 지금 메인 세션은 grep 결과 수백 줄을 본 적이 없습니다. explorer가 자기 컨텍스트에서 다 처리하고 요약만 보낸 것입니다.

### 시도해보기 2 — 두 개를 동시에 띄워 "병렬" 확인하기

> 사전 준비: 시도하기 전에 [README.md](../../../README.md)나 아무 파일을 한 줄 수정해 두세요. (그래야 reviewer가 검토할 게 생깁니다.)

1. 메인 채팅창에 입력:
   > **"explorer로 이 레포의 출시 타임라인이 어디에 있는지 찾고, 동시에 reviewer로 현재 변경사항 리뷰도 받아봐줘"**
2. UI에 **두 개의 Sub-agent 실행 표시**가 동시에 뜨는 것을 확인
3. 두 결과가 (대체로 비슷한 시점에) 도착
4. **여기서 알 수 있는 것:** 두 작업을 메인이 직접 했다면 순서대로 하나씩 했을 것입니다. Sub-agent로 분리하면 동시에 돌릴 수 있고, 메인 세션은 두 요약만 받습니다.

## 기억할 핵심 포인트

- **격리 = 메인 토큰 보호.** Sub-agent가 본 raw 데이터는 메인으로 안 옵니다.
- **격리 = 부작용 분리.** 한 Sub-agent가 헤매도 메인 세션의 흐름은 깨끗하게 유지됩니다.
- **`description` 필드가 위임의 키.** Claude는 이 description을 보고 "이 일은 Sub-agent에게 넘기는 게 맞다"고 판단합니다.
- **공짜는 아닙니다.** Sub-agent를 띄우는 데도 시간·토큰이 듭니다. **읽을 게 많고 결과는 짧을 때**가 가장 큰 이득입니다.

## 다음 데모로 가는 다리

- Skill: 같은 컨텍스트, 즉시 — **편하지만 메인 토큰 다 씀**
- Sub-agent: 격리, 병렬 가능 — **격리되어 있지만 단방향(메인이 시키고 결과만 받음)**
- 다음에 볼 **Hook**은 또 다른 결입니다: AI의 판단조차 거치지 않고 **무조건 일어나야 하는 일**을 박아두는 장치예요.
