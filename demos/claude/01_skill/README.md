# 01. Skill 데모

> **Skill이란?** 반복 워크플로우를 `SKILL.md`라는 마크다운 파일로 작성해두면, Claude Code가 자동으로 인식하거나 `/이름` 슬래시 커맨드로 호출할 수 있는 "재사용 가능한 지침서"입니다. 같은 컨텍스트 안에서 즉시 실행되므로 가장 가볍고 빠른 확장 방식입니다.

## 이 데모로 배울 것

1. **Skill의 정의 위치** — `.claude/skills/<이름>/SKILL.md`
2. **두 가지 호출 방식** — 슬래시 커맨드(`/이름`) vs 자동 인식
3. **Skill의 두 결** — 메타(레포 안내) vs 실용(커밋 메시지 작성)

## 시연용 Skill 두 개

### A. [`/intro`](../../.claude/skills/intro/SKILL.md) — "이 레포 한 바퀴" 안내

처음 이 레포를 연 사람에게 무엇이 어디 있는지를 알려주는 **메타 Skill**입니다. 입문자 청중에게 "이게 슬래시 커맨드구나"를 가장 직관적으로 보여줍니다.

**시연 방법:**
1. Claude Code 채팅창에 `/intro` 입력
2. Claude가 `SKILL.md`의 지침대로 출시 타임라인 + 시연 순서를 정리해 답변
3. 발표자: "방금 이 답변은 Claude가 즉석에서 만든 게 아니라, 우리가 [.claude/skills/intro/SKILL.md](../../.claude/skills/intro/SKILL.md)에 미리 적어둔 지침을 따라간 결과입니다"라고 설명

### B. [`/commit-msg`](../../.claude/skills/commit-msg/SKILL.md) — 커밋 메시지 제안

현재 변경사항을 보고 컨벤셔널 커밋 형식의 한국어 커밋 메시지를 제안하는 **실용 Skill**입니다. 평소에 쓸 법한 도구라 "왜 이게 유용한지"가 바로 와닿습니다.

**시연 방법:**
1. 아무 파일이나 변경 후 저장 (예: `README.md` 한 줄 추가)
2. Claude Code 채팅창에 `/commit-msg` 입력
3. Claude가 git diff를 확인 → 의도 파악 → 컨벤셔널 형식으로 메시지 후보 제시
4. 발표자: "Claude가 매번 '커밋 메시지 만들어줘'라고 부탁하지 않아도 같은 품질로 답하도록, 한 번 써둔 지침이 계속 재사용됩니다"라고 설명

## 발표 흐름 (5분 분량)

| 시간 | 내용 |
|---|---|
| 0:00 | "Skill은 Claude에게 한 번만 가르쳐두면 계속 재사용되는 지침서입니다" |

| 1:30 | [.claude/skills/intro/SKILL.md](../../.claude/skills/intro/SKILL.md) 열어서 frontmatter(`name`, `description`)와 본문이 어떻게 생겼는지 보여주기 |
| 2:30 | `/commit-msg` 시연 → 실용 케이스 보여주기 |

## Skill의 핵심 포인트 (입문자가 가져갔으면 하는 것)

- **Skill = 지침서.** 코드가 아닙니다. 한국어로 적힌 마크다운입니다.
- **`description` 필드가 트리거**입니다. Claude는 이 description을 보고 "지금 사용자 요청이 이 Skill을 호출해야 하는 상황인지" 판단합니다.
- **`/이름`은 명시적 호출, 자연어 요청은 자동 인식.** 두 길 모두 같은 Skill을 부릅니다.
- 같은 컨텍스트에서 실행되므로 **빠르지만, 무거운 작업에는 부적합**합니다 (그건 다음 데모인 Sub-agent의 영역).

## 참고: 슬래시 커맨드와 Skill의 관계

슬래시 커맨드(`/이름`)는 Claude Code 출시 초기부터 있던 인터페이스였고, 2026-04에 Skill 시스템에 통합되었습니다. 즉 지금 보는 `/intro`, `/commit-msg`는 **Skill을 호출하는 단축어**입니다. 따로 두 개를 외울 필요가 없습니다.
