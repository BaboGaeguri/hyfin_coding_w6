# 프로젝트 안내 — Claude Code에게

이 파일은 Claude Code가 세션 시작 시 자동으로 읽는 프로젝트 컨텍스트 파일입니다. 작업 시 아래 내용을 우선시해 주세요.

## 이 레포의 정체

대학교 동아리 **교육 세션용 시연 레포**입니다. 청중은 **Claude Code 입문자**이며, 발표자가 화면에 띄워놓고 실시간으로 기능을 보여주는 용도입니다. 따라서 모든 코드와 문서는 **"처음 보는 사람이 5초 안에 무슨 의미인지 감을 잡을 수 있는가"** 를 기준으로 작성합니다.

## 작성 원칙

- **데모는 짧고 자명하게.** 한 데모가 한 기능만 보여주도록. 덤으로 보여주고 싶은 게 있더라도 다른 데모로 분리합니다.
- **실제 동작하는 예제 코드**를 둡니다. 시연 중 발표자가 진짜로 실행해 청중에게 결과를 보여줄 수 있어야 합니다.
- **한국어로 설명**합니다. 코드와 식별자는 영어, 설명·README·주석은 한국어를 기본으로.
- **출시 시기와 맥락을 짚습니다.** 각 기능을 설명할 때 "왜 이 시점에 이런 기능이 추가됐는가"를 한 줄이라도 곁들이면 청중이 흐름을 잡습니다.

## 디렉토리 가이드

- `.claude/skills/` — Skill 정의. 각 폴더는 `SKILL.md` 하나로 시작합니다.
- `.claude/agents/` — Sub-agent 정의 (`.md` 파일).
- `.claude/settings.json` — Hook 등 프로젝트 단위 설정.
- `demos/01_skill` ~ `demos/05_mcp` — 각 기능별 시연 폴더. 각 폴더의 `README.md`가 그 데모의 사용 설명서입니다.
- `scripts/` — 환경 셋업이나 데모 보조 스크립트.

## 작업할 때 주의

- 데모 번호(`01_`, `02_`...)는 **시연 순서**를 따릅니다. 출시 순서와 다릅니다 (출시 순서는 README의 타임라인 표 참고).
- 새로운 데모를 만들 때는 기존 데모의 `README.md` 구조를 따라 일관성을 유지합니다.
- 청중이 자기 기기에서 따라할 수 있어야 하므로, 외부 의존성·API 키·유료 서비스가 필요한 예제는 피하거나 명확히 표시합니다.
- 코드에는 주석을 최소한으로. 대신 데모의 `README.md`에서 흐름을 설명합니다.

## 시연용 표준 흐름

발표자는 일반적으로 다음 순서를 따릅니다 — 데모를 추가/수정할 때 이 흐름을 깨지 않도록 해 주세요:

`CLAUDE.md` 설명 → Skill → Sub-agent → Hook → Agent Team → (시간 남으면) MCP
